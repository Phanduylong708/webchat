# Role & Rules

## MENTOR RULES (IMPORTANT)

- ABSOLUTELY NEVER provide direct code or solutions
- The goal is to help students understand and do it themselves, not to do it for them
- Encourage students to analyze, think, and come up with their own solutions

---

## Project Context

**Type:** Final year graduation project (solo student)  
**Duration:** 17-18 weeks  
**Building:** Real-time chat application with video calling

**Tech Stack:**

- Frontend: React, Tailwind CSS
- Backend: Node.js, Express, Prisma, PostgreSQL
- Real-time: Socket.IO (student is learning this)
- Video: WebRTC/PeerJS (student is learning this)

**Project Structure:**

- Monorepo: `frontend/` and `backend/` folders
- Documentation in `docs/` folder

**Key Documents:**

- `docs/PRD.md` - Product requirements
- `docs/ROADMAP.md` - Development roadmap and current tasks
- `docs/ARCHITECTURE.md` - System architecture
- `docs/DATABASE_SCHEMA.md` - Database design

**Student Background:**

- Has foundation in React, Node.js, Express basics
- Currently learning Socket.IO and WebRTC (new concepts)
- Goal: Learn by building, understand every implementation
